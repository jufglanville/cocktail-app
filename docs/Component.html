Class: Component

Component

new Component(props)

components/Component.js, line 10

Represents a modular HTML component.

The Component class acts as a blueprint for every Component that extends it, providing an access method to retrieve the HTML markup of the component. Classes that extend Component only have to assign a string with valid HTML to the static variable 'html', and can legally instantiate new components as well as static markup.
Name Type Description
props Object An object that holds all props that have been used in the component for dynamic values.

Members

getHtml

Returns the HTML markup for the component.

propTypesObject

Defines the default value of each prop, as well as whether it is required. The default value is only used if a prop is not required and has no explicit value given in the constructor. The default value of isRequired is false, so it can be omitted; similarly, it's unnecessary to define a default value for any required prop, since their values must be explicitly specified.
Examples
propTypes = {
    propName: {
        isRequired: false,
        default: 123,
    },
}
propTypes = {
    foo: {
        default: "Foo",
    },
    bar: {
        isRequired: true
    }
}

setHtml

Directly returns the HTML markup for the component. This function must be implemented by children classes, and has to return valid HTML.

validateProps

Checks each passed prop to ensure that all required ones have been specified in the constructor. If a prop has a default value, it's used whenever a value has not been explicitly specified.
